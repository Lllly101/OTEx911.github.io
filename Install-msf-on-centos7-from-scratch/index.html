Install msf on centos 7 from scratch

Posted on 2017-11-28 | In Penetration Test

In the penetration test, you must meet the scene where you need a host with public ip addr for reversing shell. Let’s take the following instructions to install msf on centos 7 from scratch.

It’s a must for us to know a little about the Ruby because the MSF framework is written with Ruby. So the first thing is to install the latest Ruby on your vps.

There are several ways to install Ruby:

  1. When you are on a UNIX-like operating system, using your system’s package manager is the easiest way of getting started. However, the packaged Ruby version usually is not the newest one.

  2. Installer can be used to install specific or multiple Ruby version. There is also an installer for Windows.

  3. Managers help you to switch between multiple Ruby installations on your system.

  4. And finially, you can also build Ruby from source

Here I will show you the steps to install Ruby with managers, there are serveral managers to choose.

  1. chruby

  2. rbenv

  3. rvm

  4. uru

I like use the RVM manager to install Ruby for convenience. Just type the following the command on your VPS(You can always get the command from rvm ).

For all in one installation append --rails or --ruby or --ruby=1.9.3. I added the param —ruby and then I used the rvm -v and rvm list command to check the version.

rvm -v

rvm 1.29.3 (master) by Michal Papis, Piotr Kuczynski, Wayne E. Seguin [https://rvm.io]

rvm list

ruby-2.4.1 [ x86_64 ]

Now, I copied the MSF from github.

git clone https://github.com/rapid7/metasploit-framework.git

When changing directory to the metasploit-framework, it prompts that I should upgrade the ruby to the latest version (This is always dynamic). The next step is to install bundler.

gem install bundler

The last step is to install MSF, just type bundler install. However it will fail during the installation. And you can always get the solutions from here. Here are the errors when I execute bundler install.

pcaprub.c:11:18: fatal error: pcap.h: No such file or directory

Error: Can’t find the ‘libpq-fe.h header

Those are denpencies problems, all we need to do is install them.

yum install -y postgresql-devel libpcap-devel

Now bundler install again. Everything will be done.

Happy weekday.

Summary

What’s the relation between the rvm, gem and bunlder. If you are familiar with NodeJs, you must know it. The rvm is the same as nvm just for different Ruby/Node versions installation, the gem is the same as npm for package manager. In Node, there is always a file named package.json in project, when you execute npm install then it will install all the dependencies described in the package.json. So as the bunlder, the only difference is that you need to install bundler frist with gem install bundler then execute bundler install .
