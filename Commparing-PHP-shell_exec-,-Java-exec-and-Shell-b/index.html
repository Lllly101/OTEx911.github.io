Commparing PHP shell_exec , Java exec and Shell backslash

Posted on 2017-11-28 | In Penetration Test

Command injection vulnerabilities allow an attacker to inject arbitrary system commands into an application. The command executes at the same privilege as the web application.

There are serveral ways to execute command in PHP.

  1. shell_exec() return the whole output as string
  2. exec() always return the last line of the output
  3. execution operation(``) the same to shell_exec()
  4. system() execute external program and return the last line of the output

In Java, you can only use the Runtime.getRuntime().exec() . Execute multiple commands in Java.

The symbol `` in shell is deprecated and it’s encouraged to use $(command). Bash performs the expansion by executing command in a subshell. Here is the more detail about it.

When the old-style backquote form of substitution is used, backslash retains its literal meaning except when followed by $, `, or . The first backquote not preceded by a backslash terminates the command substitution. When using the $(command) form, all characters between the parentheses make up the command; none are treated specially.

Command substitutions may be nested. To nest when using the backquoted form, escape the inner backquotes with backslashes.
