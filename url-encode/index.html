聊聊编码

Posted on 2017-05-24 | In Others

愿你走出半生，归来仍是少年

编码的相关知识一直以来是计算机的基础。信息的传递必然是建立在一定规则之上。数字、文字和自然语言一样，都是信息的载体。无论是生活中的方言、国际的英语还是计算机中的二进制，都有一套自己的规则。

由来

编码是令人头疼的问题，所以必须搞懂它。如果一知半解，编码问题就会时不时的跳出来，妨碍前行。

字符集、字符编码

字母表由26个字母组成，每个字母都有对应的符号和序号。字符集是啥，在我看来就是一堆符号的集合或表。比如ASCII码，128个字符的集合；Unicode，N多字符的合集。集合有了，那么排个序吧，方便查找。于是就有了A对应的ascii码为65，B为66。接着，怎么传输呢，全部转为二进制呗。如果只是传输ascii码中的128个字符，一个字节完全够用，当需要传输超过128个字符以外的字符时，那至少需要两个字节了。

UTF-8和Unicode

简单来说UTF-8就是Unicode的一种编码方案而已，除此还有UTF-7、UTF-16等。编码是为了数据传输，解码是给人看。

UTF-8编码为变长编码。最小编码单位（code unit）为一个字节。一个字节的前1-3个bit为描述性部分，后面为实际序号部分。

  • 如果一个字节的第一位为0，那么代表当前字符为单字节字符，占用一个字节的空间。0之后的所有部分（7个bit）代表在Unicode中的序号。
  • 如果一个字节以110开头，那么代表当前字符为双字节字符，占用2个字节的空间。110之后的所有部分（5个bit）加上后一个字节的除10外的部分（6个bit）代表在Unicode中的序号。且第二个字节以10开头
  • 如果一个字节以1110开头，那么代表当前字符为三字节字符，占用2个字节的空间。110之后的所有部分（5个bit）加上后两个字节的除10外的部分（12个bit）代表在Unicode中的序号。且第二、第三个字节以10开头
  • 如果一个字节以10开头，那么代表当前字节为多字节字符的第二个字节。10之后的所有部分（6个bit）和之前的部分一同组成在Unicode中的序号。

URL的utf-8

在百度搜索输入找字进行搜索时，仔细观察url会发现找被编码成%E6%89%BE。这种编码方式被称为百分号编码（URL Encoding）。URL默认编码是US-ASCII，当输入找时，US-ASCII中并没该字符，便到Unicode去匹配，然后对其代码点进行utf-8编码，最后进行百分号编码得到%E6%89%BE。

URL中的不安全字符:

  1. 空格: –> %20
  2. 井号: # –> %23
  3. 单引号: \’ –> %27
  4. 双引号: \” –> %22
  5. 尖角号: < –> %3c > –> %3e
  6. 百分号: % –> %25
  7. 引用符: & –> %26
  8. 括号: ( –> %28 ) –> %29

RFC3986和RFC1738分别记录了URI和URL的详细说明。有兴趣的可以参考Reference。
